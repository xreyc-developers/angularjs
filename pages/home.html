Frontend
VueJS

This is my favorite among the three frameworks/library for building SPA not just because of its simplicity but also because it is progressive and easy to maintain. Its syntax looks like AngularJS and ReactJS mixed which makes me love this framework.

I use Vuex for state management because it blends well with VueJS than Redux, maybe because it takes advantage by building it based on VueJS core.

How Vuex works?

Based on my understanding and experience with Vue + Vuex, this is how the data flows inside of vuex library, where components only trigger an action and get data via getters. Actions don't care about the state, it only deals with the business logic where calculations, API and other operations are involved.

ReactJS

ReactJS basically is an open-source JavaScript library which is used for building user interfaces specifically for single-page applications. It’s used for handling view layer for web and mobile apps. React also allows us to create reusable UI components.

How Redux works?

Redux works just like Vuex, however, they have a different process of handling the state. State and actions are handled by the reducer, it takes a new set of data from the action and using state data as the initial data and finally updates the state that can be accessed everywhere in the application.

NodeJS

Based on my understanding, NodeJS is a runtime environment for running JavaScript. It enables running JavaScript code outside a browser by using the Google Chrome's V8 engine. Since it runs JavaScript as a backend application, it enables running queries in an asynchronous fashion means non-blocking.

This is my favorite among the three backend services here because you can implement codes in a less opinionated way. Based on the implementation I’ve shown in the picture, I refactor the files to make the application more maintainable.
